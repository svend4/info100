## 13.1. СУПЕРПРОЕКТ 1: «НейроОС» — Операционная система для ИИ-жизни

**Темы:** #13, #14, #41, #43, #46, #53, #56, #72, #74, #76, #77, #85, #87

### Архитектура

```
┌─────────────────────────────────────────────┐
│              UI / Фронтенд                  │
│   (PWA + Electron/Tauri, React, WebSocket)  │
├─────────────────────────────────────────────┤
│         Семантический автобус (#85)          │
│   (gRPC / NATS / Redis Streams)             │
├──────────┬──────────┬───────────────────────┤
│ Микро-   │ RAG-     │ Инфо-Бонсай          │
│ агенты   │ движок   │ (#46)                 │
│ (#13)    │ (#11)    │ Пирамида знаний (#41) │
├──────────┴──────────┴───────────────────────┤
│      Локальный ИИ-слой (Ollama / llama.cpp) │
│      + облачный fallback (OpenAI / Claude)  │
├─────────────────────────────────────────────┤
│   Хранилище: SQLite + ChromaDB + MinIO      │
│   Sync: CRDTs (Yjs) / Syncthing            │
└─────────────────────────────────────────────┘
```

### Стек технологий

| Слой | Технологии | Почему |
|------|-----------|--------|
| **Фронтенд** | React + TypeScript, Tauri (десктоп), PWA (мобильные) | Tauri легче Electron в 10x, PWA — кроссплатформа |
| **Бэкенд / ядро** | Python 3.12+ (FastAPI), Rust (критичные модули) | Python — экосистема ML; Rust — скорость ядра |
| **Шина данных** | NATS JetStream | Легковесный pub/sub, работает оффлайн, <10 МБ |
| **Микроагенты** | LangGraph + CrewAI | Оркестрация цепочек агентов с памятью |
| **Локальный ИИ** | Ollama + llama.cpp + GGUF-модели | Оффлайн-работа, квантизованные модели 1-8 Bit |
| **RAG-движок** | LlamaIndex + ChromaDB | Зрелая экосистема, локальные эмбеддинги |
| **Хранилище** | SQLite (мета), ChromaDB (векторы), MinIO (файлы) | Всё локальное, без внешних зависимостей |
| **Синхронизация** | Yjs (CRDTs) + Syncthing | Оффлайн-first, конфликтоустойчивая синхронизация |
| **Конфигурация** | YAML (пользовательские пайплайны, #53) | Человекочитаемый формат для инфоконвейеров |
| **CI/CD** | GitHub Actions, Docker | Стандарт отрасли |

### Минимальная команда для MVP

| Роль | Чел. | Задачи |
|------|------|--------|
| Fullstack-разработчик (Python/React) | 2 | Ядро, UI, API |
| ML-инженер | 1 | Интеграция моделей, RAG, агенты |
| DevOps / Infra | 1 | Сборка, Tauri, CI/CD |
| **Итого** | **4** | **MVP за 4-5 месяцев** |

### Оценка затрат на MVP
- Зарплаты (4 чел. × 5 мес.): **€80-120K** (Европа) / **$40-60K** (Азия)
- Инфраструктура (серверы, API): **€2-5K/мес.**

- **Итого MVP: €100-150K**

## 13.7. ОБЩИЕ ПРИНЦИПЫ ВЫБОРА СТЕКА

1. **Оффлайн-first**: везде, где возможно, локальные модели (Ollama, llama.cpp, Whisper.cpp)
2. **Open-source приоритет**: минимизация vendor lock-in (PostgreSQL > Firebase, n8n > Zapier)
3. **Минимализм**: SQLite вместо MongoDB, NATS вместо Kafka, Meilisearch вместо Elasticsearch
4. **Python как lingua franca**: ML-экосистема, быстрое прототипирование, огромное сообщество
5. **Rust/C для критичных путей**: ядро НейроОС, бортовой ИИ Forth-Роя
6. **TypeScript для интерфейсов**: единый язык для веб и мобильных (через React Native / Next.js)
