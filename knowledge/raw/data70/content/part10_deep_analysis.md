## 10.1. ПРИОРИТИЗАЦИЯ ПО РЕАЛИЗУЕМОСТИ (ТОП-25 ТЕМ)

**Методология оценки:**
- **Готовность технологии (ГТ):** 1 = фундаментальное R&D, 5 = готовые инструменты на рынке
- **Рыночный потенциал (РП):** 1 = нишевый, 5 = массовый глобальный рынок
- **Сложность реализации (СР):** 1 = крайне сложно, 5 = реализуемо одним разработчиком

## 10.2. MVP ДЛЯ ТОП-10 (КОНКРЕТНЫЕ ПРЕДЛОЖЕНИЯ)

---

### MVP #1. Micro-ИИ агенты для автоматизации документооборота
**Итоговый балл: 14**

**Что строить:** CLI/десктоп-приложение, которое:
- Мониторит указанную папку (входящие документы)
- Автоматически классифицирует файлы (счета, договоры, письма, отчёты)
- Извлекает ключевые поля (дата, сумма, контрагент, срок)
- Перемещает в структурированные папки
- Генерирует ежедневную сводку

**Стек:** Python 3.12 + Ollama (Llama 3.1 8B / Mistral 7B) + LangChain + Watchdog (мониторинг FS) + SQLite (метаданные) + Textract/PyMuPDF (PDF) + python-docx

**Трудозатраты:** 3-4 недели (один разработчик)

**Первый шаг:** Написать скрипт классификации PDF-файлов через Ollama API — загрузить 10 документов разных типов, добиться точности классификации >90%.

---

### MVP #2. RAG для корпоративных данных (Google Drive)
**Итоговый балл: 14**

**Что строить:** Веб-приложение с чатом, подключённое к Google Drive / локальной папке:
- Индексация документов (PDF, DOCX, Google Docs, Sheets)
- Векторный поиск по содержимому
- Ответы с указанием источника (цитата + ссылка на файл)
- Инкрементальное обновление индекса

**Стек:** Python + FastAPI + ChromaDB / Qdrant + sentence-transformers (e5-large) + Ollama (Mistral 7B) + Google Drive API + React (фронтенд) + Docker

**Трудозатраты:** 4-5 недель (один разработчик)

**Первый шаг:** Написать пайплайн: загрузка 50 файлов из папки → чанкинг → эмбеддинги → ChromaDB → тестовые запросы через CLI.

---

### MVP #3. ИИ-автоматизация e-commerce через Make.com
**Итоговый балл: 14**

**Что строить:** Набор сценариев Make.com (5-7 штук):
1. Парсинг каталога поставщика → генерация описаний через GPT-4o mini
2. Автоматическая публикация товаров в Shopify/WooCommerce
3. Генерация постов для соцсетей
4. Автоответы на типовые вопросы клиентов
5. Еженедельная аналитика продаж

**Стек:** Make.com + OpenAI API (GPT-4o mini) + Shopify API + Telegram Bot API + Google Sheets

**Трудозатраты:** 2-3 недели (один разработчик)

**Первый шаг:** Создать один сценарий Make.com: CSV с товарами → ChatGPT (описание + SEO) → Google Sheet с результатом. Проверить на 20 товарах.

---

### MVP #4. Личный ИИ-инфоконвейер (YAML + GPT)
**Итоговый балл: 14**

**Что строить:** Python-утилита с YAML-конфигурацией:
- Мониторинг списка URL по расписанию
- Детектирование изменений (diff)
- Скачивание новых PDF/документов
- Генерация саммари через LLM
- Отправка уведомлений в Telegram

**Стек:** Python + requests/playwright + BeautifulSoup + PyYAML + Ollama/OpenAI API + python-telegram-bot + cron/APScheduler + SQLite

**Трудозатраты:** 2-3 недели (один разработчик)

**Первый шаг:** Скрипт мониторинга одного сайта: скачивание страницы → детектирование изменений → саммари через Ollama → уведомление в Telegram.

---

### MVP #5. Локальные нейросети оффлайн (улучшенный интерфейс)
**Итоговый балл: 13**

**Что строить:** Десктоп-приложение поверх Ollama с функциями, отсутствующими в Open WebUI:
- Управление «проектами» (каждый проект = своя системная инструкция + набор файлов + история)
- Drag & drop файлов для контекста
- Автоматическое переключение моделей по задаче (кодинг → CodeLlama, текст → Mistral)
- Экспорт/импорт сессий
- Оффлайн-индексация локальных файлов (мини-RAG)

**Стек:** Electron / Tauri + Svelte/React + Ollama API + LanceDB (лёгкая векторная БД) + sentence-transformers (ONNX)

**Трудозатраты:** 5-6 недель (один разработчик)

**Первый шаг:** Tauri + Svelte приложение с базовым чатом через Ollama API + управление проектами (CRUD через JSON-файлы).

---

### MVP #6. Bytebot / ИИ-автоматизация рабочего стола
**Итоговый балл: 13**

**Что строить:** Расширение/форк Bytebot с фокусом на конкретную нишу — бухгалтерия малого бизнеса:
- Запись макросов через демонстрацию (пользователь показывает → система учится)
- Автоматическое заполнение форм в 1С/SAP/Excel
- Экспорт данных из одной системы в другую
- Расписание автоматических задач

**Стек:** Python + Playwright / PyAutoGUI + Bytebot (fork) + Ollama (для понимания интерфейса) + Docker + SQLite

**Трудозатраты:** 6-8 недель (один разработчик)

**Первый шаг:** Запустить Bytebot в Docker, добиться автоматизации одного бизнес-процесса: заполнение формы в веб-приложении на основе данных из CSV.

---

### MVP #7. 105 сценариев цифровой заботы (платформа)
**Итоговый балл: 13**

**Что строить:** Telegram-бот + веб-дашборд с каталогом автоматизаций:
- Каталог из 20+ готовых сценариев (здоровье, финансы, семья, дом)
- Пошаговый визард для настройки каждого сценария
- Интеграция: Google Calendar, Google Sheets, банковские уведомления (парсинг email/SMS)
- Ежедневные/еженедельные отчёты в Telegram

**Стек:** Python + aiogram (Telegram) + Make.com (для сложных сценариев) + Google APIs + FastAPI + PostgreSQL + React (дашборд)

**Трудозатраты:** 5-6 недель (один разработчик, старт с 10 сценариев)

**Первый шаг:** Telegram-бот с 3 сценариями: (1) напоминание о приёме лекарств, (2) трекинг расходов из SMS, (3) утренняя сводка погоды+календарь.

---

### MVP #8. Инфо-Бонсай — персональная база знаний с ИИ
**Итоговый балл: 12**

**Что строить:** Браузерное расширение + бэкенд:
- Расширение: сохранение страниц/выделенных фрагментов одним кликом
- Автоматическое тегирование и классификация через LLM
- Векторный поиск по сохранённым материалам
- Генерация связей между записями (граф знаний)
- Q&A по личной базе

**Стек:** Chrome Extension (JS/TS) + FastAPI + ChromaDB + Ollama + PostgreSQL + React (дашборд) + Docker

**Трудозатраты:** 6-8 недель (один разработчик)

**Первый шаг:** Chrome Extension: кнопка «Сохранить» → отправка текста страницы на бэкенд → автотегирование через Ollama → сохранение в ChromaDB → поиск через CLI.

---

### MVP #9. B2B нейросеть (NeuroB2B)
**Итоговый балл: 12**

**Что строить:** SaaS-платформа для малого B2B-бизнеса:
- Загрузка шаблонов КП и прайсов
- Генерация коммерческих предложений под конкретного клиента
- Анализ входящих контрактов (риски, ключевые условия)
- CRM-интеграция (парсинг email → карточки контактов)

**Стек:** Python + FastAPI + Next.js + OpenAI API / Ollama + PostgreSQL + Redis + Celery + Docker

**Трудозатраты:** 8-10 недель (один разработчик)

**Первый шаг:** Скрипт генерации КП: шаблон DOCX + JSON с данными клиента → GPT-4o mini → готовый DOCX. Тест на 10 реальных случаях.

---

### MVP #10. DAT для RAG — динамический гибридный поиск
**Итоговый балл: 12**

**Что строить:** Python-библиотека (pip-пакет):
- Гибридный поиск BM25 + dense embeddings
- Автоматическая настройка весов alpha для каждого запроса через классификатор
- Бенчмарк на стандартных датасетах (BEIR)
- Простой API: `search(query, documents) → results`

**Стек:** Python + rank_bm25 + sentence-transformers + scikit-learn (классификатор типа запроса) + FAISS + pytest

**Трудозатраты:** 3-4 недели (один разработчик)

**Первый шаг:** Реализовать базовый гибридный поиск (BM25 + cosine similarity) с фиксированным alpha=0.5 на 1000 документах, измерить метрики.

## 10.3. ДОРОЖНАЯ КАРТА РЕАЛИЗАЦИИ

### Принципы построения дорожной карты:
1. **Зависимости:** некоторые проекты используют результаты других
2. **Нарастание сложности:** от простых скриптов к платформам
3. **Переиспользование компонентов:** RAG-модуль, Ollama-интеграция, Telegram-бот

---

### ФАЗА 1: ФУНДАМЕНТ (месяцы 1-2)
*Цель: создать базовые компоненты, которые будут переиспользоваться*

```
Неделя 1-3:  MVP #4 — ИИ-инфоконвейер (YAML + GPT)
             ├── Навык: работа с Ollama API, парсинг, Telegram-бот
             └── Компонент: модуль мониторинга + LLM-саммари

Неделя 3-6:  MVP #10 — DAT для RAG (библиотека)
             ├── Навык: векторный поиск, BM25, эмбеддинги
             └── Компонент: гибридный поисковый движок

Неделя 5-8:  MVP #1 — Micro-ИИ агенты (документооборот)
             ├── Использует: Ollama API (из #4)
             └── Компонент: классификатор документов, экстрактор полей
```

**Результат Фазы 1:** Три рабочих инструмента + переиспользуемые модули (LLM-клиент, парсер документов, векторный поиск, Telegram-бот).

---

### РЕКОМЕНДАЦИИ ПО ПОРЯДКУ ЗАПУСКА

1. **Начинать с MVP #4 (Инфоконвейер)** — самый простой, даёт немедленный результат и навыки работы с LLM API.

2. **Параллельно делать MVP #10 (DAT для RAG)** — библиотека, которая станет конкурентным преимуществом всех последующих продуктов.

3. **MVP #3 (E-commerce автоматизация)** можно запускать параллельно с любой фазой — он использует no-code (Make.com) и не зависит от других компонентов.

4. **MVP #9 (NeuroB2B)** — самый сложный, но и самый коммерчески перспективный. Запускать только после накопления всех компонентов из Фаз 1-2.

5. **Долгосрочные проекты** (Compute-in-Memory, LCM, нейроморфные чипы) — отслеживать развитие, участвовать в open-source сообществах, но не инвестировать собственные ресурсы в хардверные R&D.
